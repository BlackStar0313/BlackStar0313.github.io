独立游戏开发日志Sprint1

发布的第一篇文章收到的很多留言和关注，备受鼓舞，给了我很强烈的反馈，我预计的开发周期为一年，时间很长且只有一人，暂定一周一个sprint，一篇开发日志，总结我这周的开发进度，和分享开发心得，希望我开发过程可以给需要的伙伴提供一些有价值的内容，也希望可以获得不同的声音给予我反馈，得到学习和及时调整，以不至于自己闭门造车。

由于之前所有精力毫无保留的留给了前公司，现在自己做所有内容都得从0开始了，开发环境就够喝一壶的了，前几个sprint预计需要花多一点时间先做一些脚手架搭建，实际游戏逻辑开发会往后放放。

sprint1完成的内容：

  • 创建了unity项目，基于unity 6000.0.58f2 (我的游戏不是肉鸽类游戏，预计实例不会特别多，暂不考虑用DOTS做，代码动态更新的也不支持新一点的DOTS)
  • 用AI做了几个辅助开发的Editor工具。
  • cursor与unity联调可debug. 没错，真的是从0开始。 cursor中Debug用DotRush插件 unity加cursorIDE包看我的这篇文章 https://blackstar0313.github.io/docs/Unity/%E5%9C%A8Unity%E4%B8%AD%E4%BD%BF%E7%94%A8Cursor
  • 引入hybridCRL + yooassets做代码和资源的热更新。测试了打的本地mac包和android包。资源放在本地的nginx服务和阿里云oss中远程拉取，都ok。
  • 手工打出了第一个Android包，并简单优化了下包体从35兆减到31兆，目前只加了几个额外的pacakge，由于我需要IL2CPP + URP + ARM64，不知道这样大小的apk算ok的不？

  • 确定项目管理工具- Notion之前用过Tower,Asana,飞书，Tower很多年前用的，现在似乎卖给别人不维护了，其他的还到处看了看，别的jira,腾讯的tapd,等等好多，全都是免费版本功能少，付费我一个人又没必要，我比较在意有timeline甘特图或者日历可以总览一下task进行调整，开发到后面琐碎的内容多了没有这个感觉会要命。最后选用了Notion，一个人够用，并且非常自由，想要什么样子可以自己拼成什么样子，我用他做项目管理和scrum记录，也用作项目的知识库，可以将task互相关联到对应的sprint，sprint中可以看到所有task和进度，有看板，有甘特图timeline,可以自定义属性，属性可以跟excel一样写公式逻辑实现不同展示。

热更和打包流程没做细化，先跑通下流程，可以确定目录结构，尽量减少后期调整，DevOps流程等后面做差不多了再整吧，先都手工操作，IOS包没打，还没买开发者账号没证书，等开发差不多了再买账号整。

一些工具调研和选择花了不少时间，一步步来，第一个sprint感觉挺好，跟当牛马比每天能保证充足睡眠，一天也就干个五六个小时，还有时间健身自己做饭，持久战得慢慢推进。

感谢阅读，欢迎沟通互相学习
