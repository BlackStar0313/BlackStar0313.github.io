适合与AI配合的流程图Mermaid

诉求​

有时需要画一些流程图，比如战斗的流程逻辑，xmind那种脑图用来展示规划和想法比较合适，但是画一些流程逻辑相对复杂的流程图，显得有些吃力，含有流程判断 ，条件分支等，而其他drawio等可视化工具，不太方便与AI配合。而Mermaid则是一个非常适合与AI配合的流程图工具。

亲测​

给AI提供prompt，生成Mermaid代码，复制到vscode中，体验非常丝滑，我只需要尽情描述我的思路就好，不需要去关心语法图表美观什么的，AI会完善细节，并且给出的流程图高于我的预期。而且想要修改，代码很简单，随手就改了。不需要额外学习。

Mermaid​

基于 JavaScript 的流程图和图表工具，可以将类似 Markdown 的文本定义动态渲染为图表，实现流程图的创建和修改。 GitHub地址 拥有8w+star，可见其受欢迎程度。 具体Mermaid介绍： Mermaid

历史 ( 本段历史相关内容由AI生成 )​

  • 2014年
    Mermaid 项目在 GitHub 上由 Knut Sveidqvist 创建诞生，初衷是为了让开发者可以用简单的文本（类似 Markdown）的形式，快速创建流程图，并嵌入到文档或网页中。它最初只是面向内部开发者的小工具。

  • 2015~2017年
    随着 Markdown 流行，特别是在 GitHub 和开源社区内，Mermaid 引起了更多开发者关注。其基于 JavaScript 的实现，逐步实现了流程图、时序图、甘特图、类图等多种常用图表类型。

  • 2018~2019年
    随着文档自动化（如 Docs-as-Code 思想）兴起，Mermaid 逐渐被集成到更多工具链与平台，如 GitHub、GitLab、Hexo、Docusaurus、Obsidian 等，成为工程化文档写作的“标配”。Mermaid 的语法变得更加完善，社区贡献以及第三方插件也开始丰富起来。

  • 2020年以后
    Mermaid 发展迅猛，得益于 AI 辅助编程（如 ChatGPT、Copilot 等）的兴起，Mermaid 的“代码化画图”模式特别适合用 AI 生成与完善图表。GitHub、Notion、Obsidian 等主流平台都原生或间接支持 Mermaid 语法，大大降低了流程图、架构图等文档插画的门槛。2022年左右，Mermaid 成为前端、后端、运维、产品等各类人员都广泛应用的流程图工具，迎来爆发。

  • 现在
    Mermaid 已经是一款事实上的主流流程图和架构图工具。其开源、跨平台、可集成、可编程的特性让它能够和 AI 等新兴工具一起协作，大幅提升了文档可视化的效率和创造力。

Mermaid 的兴起体现了“以代码描述图表”的现代开发和协作趋势。AI 的发展更加推动了 Mermaid 在工程、产品、知识管理领域的爆火，让流程图的表达更开放、自动化、更易修改与协作。

为什么适合与AI配合​

以前不是很火，因为之前需要手动去写逻辑，需要记住一些语法（类似很久不会有人用的UML,需要记住一些逻辑和语法），对于程序来说相当于学一个简单的编程语言，更不要说其他职业了，所以不是很流行。

在AI普及的今天，这反而成了他的优势,偏程序化的图表，只需要将你的流程想法，用自然语言描述出来，让AI生成对应的Mermaid代码，然后复制到vscode中，就可以看到对应的流程图。而且他与xmind等保存后是一个二进制文件不同，mermaid本身就是一个md文件，是文本文件，可以通过git进行版本管理，修改内容也可以非常直观的看到。

Mermaid不仅仅是画图工具，更是一种结构化思维与文档自动化的工具。

目前支持的相当成熟了：​

  • GitHub / GitLab: 你在项目的 README.md 里直接写 Mermaid 代码，GitHub 会直接把它渲染成图。这对于游戏开发项目的文档管理简直是神器。
  • Notion: 全球最火的笔记软件，输入 /mermaid 就能直接画图。
  • Obsidian: 知识库软件的顶流，原生支持 Mermaid。
  • VS Code: 装个插件就能边写边预览。
  • Docusaurus: 本博客使用的网站框架，也支持 Mermaid。

示例(Docusaurus与vscode)​

在本博客下示例：Docusaurus​

安装插件

npm install @docusaurus/theme-mermaid

在docusaurus.config.ts中添加：

markdown: {
  mermaid: true,
},
themes: ['@docusaurus/theme-mermaid'],
