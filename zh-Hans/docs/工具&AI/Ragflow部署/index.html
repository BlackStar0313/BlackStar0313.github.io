Ragflow部署

本地环境​

  • 本地环境：macOS 15.2
  • 机器：Mac mini M4 32GB

官网 https://ragflow.io/docs/dev/

如果你是windows用户，或者其他 x86 架构的机器，请参考官网文档。git clone下来工程内就有预编译好的镜像，可以直接 compose到docker中使用。 如果你是M系列芯片的Mac 或其他ARM架构的机器，需要自己手动Build镜像 Image,再进行compose 。参考文档: https://ragflow.io/docs/dev/build_docker_image 如官网所说

ARM芯片架构的Mac 部署​

安装docker ,下载docker desktop https://www.docker.com/get-started/

我使用的是需要向量模型的 如果你没有梯子，修改下Dockfile 下面的NEED_MIRROR

ARG NEED_MIRROR=1 # 改成1，走国内镜像

如果可以全局🪜的话这里不需要修改。

安装下载依赖。(如果不做这部直接build镜像，会报错 )

pip3 install huggingface_hub nltk
python3 download_deps.py

这里安装相关的，如果你电脑没有做过相关的开发，或者像我一样是台新买的Mac，那么这里会有很多相关需要安装的依赖。 比如 brew pip python 等。你还需要配置对应的环境变量。这里可能需要多花点时间搞，如果不熟悉的话，配置环境变量真的是很麻烦。。。。

还有个问题，由于macOS 系统保护 Python 环境，所以需要创建虚拟环境来解决。具体如何做，看这篇文章 Mac虚拟环境

安装完依赖后，修改配置文件 .env

RAGFLOW_IMAGE=infiniflow/ragflow:v0.16.0


# Optimizations for MacOS
# Uncomment the following line if your OS is MacOS:
MACOS=1

修改配置文件 docker-compose-base.yml

....
environment:
    .....
    - ES_JAVA_OPTS=-XX:UseSVE=0
    - CLI_JAVA_OPTS=-XX:UseSVE=0
......

不修改这个的话，启动docker后 ragflow-es-01 会报错 具体可看这篇 ：https://github.com/infiniflow/ragflow/issues/4733

上述步骤完成后，就可以开始build镜像了。

docker build -f Dockerfile.deps -t infiniflow/ragflow_deps .
docker build -f Dockerfile -t infiniflow/ragflow:v0.16.0 .

可以将build好的镜像 compose 到docker中,启动server,等待其他依赖的镜像拉取。

docker compose -f docker-compose-base.yml up -d

查看server状态

docker logs -f ragflow-server

如果输出是这个 才表示所有工作都work了

Ollama 配置​

  1. 设置 Ollama 监听所有接口：
export OLLAMA_HOST=0.0.0.0:11434
ollama serve
  1. 确认监听状态：
netstat -an | grep 11434
# 应该看到 [::]:11434
  1. 在 Ragflow 中配置：
  • Base URL: http://192.168.1.18:11434
  • Model Name: deepseek-r1:32b

注意：

  • 确保 Ollama 服务持续运行

  • 如果重启系统，需要重新启动 Ollama 服务

  • 可以将环境变量添加到 ~/.zshrc 使配置永久生效
