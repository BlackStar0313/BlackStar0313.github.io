通过Gulp拉取googleSheet数值表到本地

使用服务账号访问 Google Sheets​

1. 创建项目并启用 API​

  1. 打开 https://console.cloud.google.com/ 并登录 Google 账号。
  2. 在顶部选择或新建一个项目（Project）。
  3. 进入 “API 和服务” → “启用 API 和服务”，搜索并启用 Google Drive API 与 Google Sheets API。

2. 创建服务账号并生成密钥​

  1. 在 “API 和服务” → “凭据” 中点击 “创建凭据” → “服务帐号”。
  2. 填写名称与描述，角色可暂留空；如果后续需要管理权限，可分配“项目 → 编辑者”等角色。
  3. 在新建的服务账号详情页选择 “密钥” 标签 → “添加密钥” → “创建新密钥”，类型选择 JSON，下载得到 service-account.json。

3. 赋予表格访问权限​

  1. 打开要同步的 Sheet：https://docs.google.com/spreadsheets/d/Google Sheet 的文件ID/edit。
  2. 点击右上角 “共享”，将服务账号邮箱（形如 xxxx@xxxx.iam.gserviceaccount.com）设置为 编辑者 或至少 查看者。
  3. 只有将表格共享给服务账号后，脚本才能成功读取数据。

Node/Gulp 脚本配置​

  • 将 service-account.json 放入本地 gulp 或工具目录，并确保该文件不会被提交至公开仓库。
  • 在 Node 脚本中设置 fileId = "Google Sheet 的文件ID"，调用 google.auth.GoogleAuth({ keyFile: 'service-account.json', scopes: [...] }) 使用服务账号认证。
  • 推荐最小 scopes：https://www.googleapis.com/auth/drive.readonly；如果需要写回 Drive，可改用 drive.file 或更高权限。

可选：使用 OAuth2（个人账号授权）​

  1. 在 “凭据” 页面创建 “OAuth 客户端 ID”，类型选择 “桌面应用” 或 “外部应用”。
  2. 运行 Google 官方示例完成首次授权流程，浏览器会弹窗要求登录并授予权限。
  3. 授权成功后会生成 token.json，后续脚本即可复用凭据；该方式适合本地临时脚本，CI/服务器环境仍推荐服务账号。

导出 Google Sheet 到本地​

  1. 完成以上配置后，执行对应的 Node/Gulp 任务。
  2. 使用 files.export 将 Sheet 导出为 .xlsx 或其他需要的格式并写入本地。
  3. 验证导出的数据是否符合预期，必要时加入自动化校验。
