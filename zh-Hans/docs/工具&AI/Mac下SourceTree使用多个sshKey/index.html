Mac 上生成多个 SSH 密钥并在 Sourcetree 中使用

本文将指导您如何在 Mac 上生成多个 SSH 密钥，并配置 Sourcetree 使用不同的密钥连接不同的远程仓库。

目录​

  1. 生成 SSH 密钥
  2. 配置 SSH 使用多个密钥
  3. 在 Sourcetree 中使用特定 SSH 密钥
  4. 测试 SSH 连接
  5. 常见问题解决

生成 SSH 密钥​

在 Mac 上生成 SSH 密钥非常简单。以下是生成多个 SSH 密钥的步骤：

生成第一个 SSH 密钥（RSA 类型）​

  1. 打开终端（Terminal）
  2. 输入以下命令，请将电子邮件地址替换为您的实际邮箱：
ssh-keygen -t rsa -b 4096 -C "your_email@example.com"
  1. 当提示"Enter a file in which to save the key"时，直接按回车键接受默认文件位置（~/.ssh/id_rsa）
  2. 根据提示输入密码（如果需要的话），或直接按回车键不设置密码

生成第二个 SSH 密钥（ED25519 类型）​

  1. 在终端中输入以下命令，使用另一个电子邮件地址：
ssh-keygen -t ed25519 -C "your_second_email@example.com"
  1. 当提示"Enter a file in which to save the key"时，输入一个自定义路径，例如：
/Users/yourusername/.ssh/id_ed25519_account2
  1. 根据提示输入密码（如果需要的话），或直接按回车键不设置密码

添加 SSH 密钥到 SSH 代理​

  1. 启动 SSH 代理：
eval "$(ssh-agent -s)"
  1. 添加您的 SSH 私钥到 SSH 代理：
ssh-add ~/.ssh/id_rsa
ssh-add ~/.ssh/id_ed25519_account2
  1. 验证密钥已添加：
ssh-add -l

配置 SSH 使用多个密钥​

为了让 SSH 知道应该使用哪个密钥连接到哪个服务器，需要创建或编辑 SSH 配置文件：

  1. 创建或编辑 SSH 配置文件：
nano ~/.ssh/config
  1. 添加以下配置（根据您的实际情况修改）：
# 默认 GitHub 账号（使用 id_rsa）
Host github.com
    HostName github.com
    User git
    IdentityFile ~/.ssh/id_rsa

# 第二个 GitHub 账号（使用 id_ed25519_account2）
Host github-account2
    HostName github.com
    User git
    IdentityFile ~/.ssh/id_ed25519_account2
  1. 保存并退出（按 Ctrl+O，然后按 Enter，再按 Ctrl+X）

在 Sourcetree 中使用特定 SSH 密钥​

添加新仓库时​

  1. 打开 Sourcetree
  2. 点击"New Repository"或"Clone"按钮
  3. 在 URL 字段中，使用相应的 Host 别名：
    • 对于默认账户：git@github.com:username/repository.git
    • 对于第二个账户：git@github-account2:username/repository.git

修改现有仓库的远程 URL​

  1. 在 Sourcetree 中，右键点击仓库
  2. 选择"Repository Settings"（仓库设置）
  3. 点击"Remotes"（远程）选项卡
  4. 选择要修改的远程仓库（通常是"origin"）并点击"Edit"（编辑）
  5. 修改 URL，将 github.com 改为 github-account2（对于第二个账户）
  6. 点击"OK"保存更改

或者，您可以在终端中使用以下命令：

git remote set-url origin git@github-account2:username/repository.git

测试 SSH 连接​

测试默认 GitHub 账户连接​

ssh -T git@github.com

如果成功，您将看到类似以下的消息：

Hi username! You've successfully authenticated, but GitHub does not provide shell access.

测试第二个 GitHub 账户连接​

ssh -T git@github-account2

同样，成功时会显示认证成功的消息。

常见问题解决​

权限被拒绝（Permission denied）​

如果遇到"Permission denied (publickey)"错误：

  1. 确保您已将公钥添加到 GitHub 账户
  2. 检查 SSH 配置文件是否正确
  3. 确认私钥已添加到 SSH 代理：
ssh-add -l

密钥格式错误​

确保您的 SSH 配置文件中的路径指向正确的私钥文件（没有 .pub 扩展名）。

Sourcetree 无法连接​

  1. 在 Sourcetree 中，转到"Preferences"（首选项）
  2. 选择"General"（常规）选项卡
  3. 确保"SSH Client"设置为"OpenSSH"

调试 SSH 连接​

使用以下命令查看更详细的连接信息：

ssh -vT git@github-account2

通过以上步骤，您可以在 Mac 上生成和管理多个 SSH 密钥，并在 Sourcetree 中灵活使用它们连接不同的远程仓库，方便地管理多个账户的代码仓库。
