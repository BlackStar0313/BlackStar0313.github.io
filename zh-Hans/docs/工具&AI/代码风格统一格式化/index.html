代码风格统一格式化

诉求​

  • 可以编辑器保存时自动格式化
  • 不同人在不同编辑器格式化代码风格一致
  • 不同平台下格式化代码风格一致
  • 防止以上情况导致提交时冲突
  • 规则可进入版本管理

我使用的是vscode, 它本事有自己的格式化配置，但是是在本地配置的，没有在版本库管理，而且不同ide可能有差异，所以选择换个方式。

.editorconfig + csharpier插件​

.editorconfig​

可保证规则可编辑可版本管理。

但是.editorconfig 更像“规则说明书”，真正执行格式化的是各自的格式化器（Visual Studio、Rider、dotnet format 等） 同一份 .editorconfig 在不同工具里解释方式会略有差异：例如有些规则只在 VS 生效，Rider 可能忽略，或某些新语法尚未实现，结果就会出现“几乎一样但细节不同”。

csharpier插件​

  1. 安装csharpier插件

  1. 配置csharpier插件
dotnet new tool-manifest
dotnet tool install csharpier

常见于 1) 需要跨 IDE（VS、Rider、VS Code）统一格式的团队；2) 在 CI 中保证提交始终符合规范；3) 想要确定性输出（格式化两次结果完全一致）以减少 Git diff。社区 adoption 虽不及微软官方 formatter，但在 Unity、Blazor、后端团队里逐渐有人将其作为“唯一格式化器”使用。

CSharpier 的思路是：不管 .editorconfig 写了什么，都由它自己定义固定输出，团队所有人、所有 IDE、命令行跑出来的格式完全一致。

我这里没有用.csharpierrc文件定义格式，而是将.editorconfig文件作为唯一配置文件,csharpier插件会自动读取.editorconfig文件中的配置做格式化器。

csharpier忽略规则​

.csharpierignore 文件，比如我这里不想第三方库被格式化，不想某些文件被格式化，可以这样写：

Packages/**
!Packages/com.gmf.framework
!Packages/com.gmf.framework/**
Assets/*.config

整个项目格式化​

dotnet csharpier format .

将项目应用csharpier格式化在保存文件时候自动格式化​

cmd + shift + p -> csharpier: format document

下面的 “Set as Default Formatter” 点一下，这样在保存文件时候就会自动格式化。
