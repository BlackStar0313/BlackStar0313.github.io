上下文关键字

根据上下文，有些关键字有特殊的用途，这些关键字被称为上下文关键字。但是并不是C# 的保留字。

Jqurey相关​

group into join ascending descending equals by from let on orderby select when where

add remove​

add 关键字用于定义事件调用时的自定义事件访问器. remove 关键字用于定义事件调用时的自定义事件访问器.

public class Button
{
    // 后台事件存储字段
    private EventHandler _click;
    
    // 自定义事件实现
    public event EventHandler Click
    {
        add
        {
            // 自定义添加事件处理程序的逻辑
            Console.WriteLine("添加了事件处理程序");
            _click += value;
        }
        remove
        {
            // 自定义移除事件处理程序的逻辑
            Console.WriteLine("移除了事件处理程序");
            _click -= value;
        }
    }
    
    // 触发事件的方法
    public void OnClick()
    {
        _click?.Invoke(this, EventArgs.Empty);
    }
}

// 使用示例
Button button = new Button();
button.Click += (sender, e) => Console.WriteLine("按钮被点击了");

not or and​

主要用于模式匹配上下文。

// 在模式匹配中使用
if (person is Student and not GraduateStudent) { /* ... */ }
if (shape is Circle or Rectangle) { /* ... */ }

逻辑与或 非 !, &&, || 是逻辑运算符.

alias​

用于定义别名.当需要引用同一个外部程序集名的两个版本时，可以使用别名。

/r:GridV1=grid.dll

/r:GridV2=grid20.dll

 extern alias GridV1;  
  
 extern alias GridV2;  

 using GridV1::Grid;
 using GridV2::Grid;

ascending descending​

用于排序表达式中的升序和降序。

var sortedPeople = people.OrderBy(p => p.Age, SortDirection.Ascending);
IEnumerable<string> sortAscendingQuery =
    from vegetable in vegetables
    orderby vegetable ascending
    select vegetable;

args​

并不是关键字 ， 只是一个约定俗成的命名。

public void PrintArgs(params string[] args)

await async​

用于异步编程。

public async Task<string> GetDataAsync()
{
    return await Task.FromResult("Hello, World!");
}

equals​

用于比较两个值是否相等。在join 中使用。

var query = from p in people
            join c in customers on p.Id equals c.Id
            select new { p.Name, c.Name };

get set​

访问器.

public int Age { get; set; }

value​

隐式参数用于属性和value set 访问器。

public int Age { get; set { _age = value; } }
// 或者使用表达式体
public int Age { get; set => _age = value; }

global​

全局命名空间。使用情况是当自己的命名空间与系统命名空间冲突时，可以使用全局命名空间。

// 自定义的 Console 类
public class Console
{
    public void Log(string message)
    {
        // 自定义实现
    }
    
    public void PrintWithSystemConsole()
    {
        // 使用 .NET 的 Console，而不是引用自身
        global::System.Console.WriteLine("通过.NET Console打印");
    }
}

init​

init 关键字是 C# 9.0 (2020年)引入的一个新特性，用于创建"只在初始化时可赋值"的属性。它是对属性 setter 的一种修饰，介于 set 和 readonly 之间，提供了一种新的不可变性模型。

public class Person
{
   // init 访问器的属性
   public string FirstName { get; init; }
   public string LastName { get; init; }
   public int Age { get; init; }
}
// 使用对象初始化器设置初始值
var person = new Person
{
   FirstName = "张",
   LastName = "三",
   Age = 30
};

// 初始化后不能修改
// person.Age = 31; // 编译错误!

nameof​

用于获取变量、类型、成员等的名称。是编译时表达式，不是运行时表达式。

优点：

  1. 类型安全：编译时检查，避免运行时错误。
  2. 可读性：更直观，易于理解。
  3. 可维护性：更容易修改，因为只需要修改名称，不需要修改代码。
var name = nameof(person.Name);

notnull​

是一种约束，用于约束参数不能为null。

public class Container<T> where T : notnull
{
   // T 不能是可空类型
}

partial​

用于定义分部类、分部结构体、分部接口等。

编译时，会将两个部分合并为一个完整的类定义。

public partial class Person{
   public string Name { get; set; }
}

public partial class Person{
   public int Age { get; set; }
}

public partial class Person{
    public partial string ProcessData(string data);
}

public partial class Person{
   public partial string ProcessData(string data){
       return data.ToUpper();
   }
}

record​

用于定义记录类型。记录类或者结构体。提供一种更简洁的方式定义不可变的数据类型.

对于类，直接将class 关键字替换为 record 关键字。

record 关键字为C#带来了函数式编程中常见的不可变数据类型的优势，同时保留了面向对象的特性。它特别适合作为数据容器、DTOs和领域模型中的值对象，大大简化了不可变类型的定义，并自动提供了许多有用的功能，如值相等性、with表达式和解构。

// 基本 record 声明
public record Person(string FirstName, string LastName, int Age);

// 使用 record
var person = new Person("张", "三", 30);
Console.WriteLine(person); // 输出: Person { FirstName = 张, LastName = 三, Age = 30 }
  1. 值相等
var person1 = new Person("张", "三", 30);
var person2 = new Person("张", "三", 30);

Console.WriteLine(person1 == person2); // 输出: true
Console.WriteLine(person1.Equals(person2)); // 输出: true

// 对比普通类的引用相等
var class1 = new SomeClass { Name = "测试" };
var class2 = new SomeClass { Name = "测试" };
Console.WriteLine(class1 == class2); // 输出: false
  1. 不可变性 记录默认采用 init 访问器，使其属性在初始化后不可修改：
var person = new Person("张", "三", 30);
// person.Age = 31; // 编译错误: 不能修改只读属性
  1. 非破坏性 用 with表达式创建一个已有记录的副本，并修改其中的某些属性。
// 非破坏性修改
var updatedPerson = person with { Age = 31 };
Console.WriteLine(updatedPerson); // 输出: Person { FirstName = 张, LastName = 三, Age = 31 }
  1. 结构
var person = new Person("张", "三", 30);
var (firstName, lastName, age) = person;

Console.WriteLine($"{firstName} {lastName}, {age}岁"); // 输出: 张 三, 30岁
  1. 继承
public record Student(string FirstName, string LastName, int Age, string School) : Person(FirstName, LastName, Age);
  1. 密封记录，防止被继承
public sealed record Teacher(string FirstName, string LastName, int Age, string School) ;
  1. 结构体记录
public record struct Point(int X, int Y);

with​

与record 一起使用，用于创建一个已有记录的副本，并修改其中的某些属性。

var updatedPerson = person with { Age = 31 };

unmanaged​

用于约束类型为非托管类型。

unsafe public class UnmanagedClass<T>  where T : unmanaged
{
    public T Value;
}

var​

用于隐式类型化局部变量。

var name = "张三";

when​

  1. swtich 语句中的 case guard
switch (condition)
{
    case int i when i > 0:
        Console.WriteLine("大于0");
        break;
}
  1. swwitch 表达式中的 case guard
static Point Transform(Point point) => point switch
{
    { X: 0, Y: 0 }                    => new Point(0, 0),
    { X: var x, Y: var y } when x < y => new Point(x + y, y),
    { X: var x, Y: var y } when x > y => new Point(x - y, y),
    { X: var x, Y: var y }            => new Point(2 * x, 2 * y)
};

3.try catch 中的 catch guard

try
{
    // 可能抛出异常的代码
}
catch (Exception e) when (e.Message.Contains("特定错误"))
{
    Console.WriteLine("特定错误");
}
catch (Exception e) when (e.Message.Contains("其他错误"))
{
    Console.WriteLine("其他错误");
}

yeild​

实现迭代器模式，运行方法在生成序列元素时保持状态，按需延迟计算元素。 yield 关键字用于两种主要语句：

  • yield return：返回序列中的下一个元素
  • yield break：终止迭代器的执行

当使用 yield 关键字时，编译器会自动生成一个实现了 IEnumerable<T> 和 IEnumerator<T> 接口的状态机类，这个类跟踪迭代的当前位置。

one more thing​

  1. 字段与属性区别
特性字段 (不使用 get; set;)属性 (使用 get; set;)
封装性直接暴露内部数据提供数据访问的控制层
验证逻辑无法添加可以在 get/set 中添加
计算值不支持可以返回计算值
通知机制不支持可以实现属性变更通知
序列化可能导致问题更好的序列化支持
接口实现不能用于接口可以实现接口属性
绑定支持有限支持完全支持数据绑定
  1. init 与 set 与 readonly 区别
特性get; set;get; init;get;
初始化器中设置✓✓只能在声明时
构造函数中设置✓✓✓
创建后修改✓✗✗
修改时的错误无编译时错误编译时错误
适用场景可变数据初始化后不可变完全不可变/派生计算
  1. yeild 与 await 区别
特性yieldawait
主要功能创建迭代器，生成序列元素等待异步操作完成
引入版本C# 2.0 (2005)C# 5.0 (2012)
返回类型IEnumerable<T>, IEnumerator<T>Task, Task<T>, ValueTask<T>
方法修饰普通方法需 async 修饰
执行模型延迟执行、同步异步执行
