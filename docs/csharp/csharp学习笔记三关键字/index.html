关键字

和其他语言一样，C# 也有一些关键字，这些关键字有特殊的用途，不能作为标识符使用。

abstract​

abstract 关键字用于定义抽象类/方法/属性/事件/索引器.抽象类不能被实例化，抽象方法没有实现，必须在派生类中实现。

abstract class Animal
{
    public abstract void MakeSound();
}

as​

as 关键字用于将一个对象转换为指定类型或可为 null 的类型。如果转换失败，返回 null。

var animal = new Animal();
var dog = animal as Dog;

base​

base 关键字用于访问基类中的成员。类似于 typescript 中的 super

class Animal
{
    public void Eat()
    {
        Console.WriteLine("Animal is eating");
    }
}

class Dog : Animal
{
    public void Bark()
    {
        base.Eat();
        Console.WriteLine("Dog is barking");
    }
}

explicit implicit​

显式转换 explicit 和隐式转换 implicit

public struct Temperature {
    private decimal degrees;

    // 隐式转换：可以直接赋值
    public static implicit operator decimal(Temperature t) {
        return t.degrees;
    }

    // 显式转换：需要强制类型转换
    public static explicit operator int(Temperature t) {
        return (int)t.degrees;
    }
}

// 使用示例
Temperature temp = new Temperature { degrees = 23.5M };
decimal d = temp;        // 隐式转换，无需显式转换
int i = (int)temp;      // 显式转换，需要强制转换

fixed​

固定内存 , 用于在栈上分配内存, 防止垃圾回收器重新定位可移动变量，固定变量地址语句在持续时间内不变。

unsafe struct Point
{
    public int x;
    public int y;
}

unsafe static void Main()
{
    Point p = new Point();
    p.x = 10;
    p.y = 20;
    fixed (int* ptr = &p.x)
    {
        Console.WriteLine(*ptr);
    }
}

lock​

lock语句获取给定对象的互斥锁，执行语句块，然后释放锁。保证同一时间只有一个线程访问该对象。

lock (obj)
{
    // 访问共享资源
}

lock的对象为引用类型， 该类型要避免这几种情况

  • this , 调用是也可能锁定 this
  • null , 不能锁定 null
  • 值类型， 不能锁定值类型
  • 字符串， 不能锁定字符串,可能是暂存的
  • Type 实例，实例是可以通过 typeof 或者反射获取.

可重载运算符

类别运算符说明
一元运算符+x, -x, !x, ~x, ++, --, true, falsetrue 和 false 必须一起重载
二元运算符+, -, *, /, %, &, `, ^`
移位运算符<<, >>, >>>位移操作
比较运算符==, !=, <, >, <=, >=必须成对重载

不可重载运算符

运算符备选方法
x && y，x || y重载 true 和 false 运算符以及 & 或 | 运算符。有关详细信息，请参阅用户定义的条件逻辑运算符。
a[i]，a?[i]定义索引器。
(T)x定义可由强制转换表达式执行的自定义类型转换。有关详细信息，请参阅用户定义转换运算符。
+=, -=, *=, /=, %=, &=, |=, ^=, <<=, >>=, >>>=重载相应的二元运算符。例如，重载二进制 + 运算符时，会隐式重载 +=。
^x、x = y、x.y、x?.y、c ? t : f、x ?? y、??= y无。
x..y, x->y, =>, f(x), as, await, checked, unchecked, default, delegate, is, nameof, new, sizeof, stackalloc, switch, typeof, with无。

in​

泛型修饰符 指定类型参数是逆变的，使用的类型比泛型参数指定的类型派生程度更小。

 // Contravariant interface.
interface IContravariant<in A> { }

// Extending contravariant interface.
interface IExtContravariant<in A> : IContravariant<A> { }

// Implementing contravariant interface.
class Sample<A> : IContravariant<A> { }

class Program
{
    static void Test()
    {
        IContravariant<Object> iobj = new Sample<Object>();
        IContravariant<String> istr = new Sample<String>();

        // You can assign iobj to istr because
        // the IContravariant interface is contravariant.
        istr = iobj;
    }
}

override​

重写基类中的方法, 一些限制条件

  • 方法签名必须与基类中的方法签名完全相同
  • 访问修饰符必须与基类中的方法访问修饰符完全相同
  • 只读类型支持协变返回类型
  • 要使用 override 关键字， 方法必须被 virtual abstract 或 override 修饰

可变参数， 用于方法参数， 可以传递任意数量的参数。 c# 13 之前 参数必须是一维数组， c# 13 之后 有其他类型支持 

public void Print(params string[] args)
{
    foreach (var arg in args)

        Console.WriteLine(arg);

readonly​

只读修饰符， 用于修饰字段， 表示该字段只能在声明时或构造函数中初始化， 不能在其他地方修改。 当修饰值类型时，该值不可变。 当修饰引用类型时，该引用不可变， 但是引用指向的对象可以改变。

public readonly int a = 1;

还可修饰实例成员，不会修改结构的状态

public struct Point
{
public double X { get; set; }
public double Y { get; set; }
// readonly 方法：保证不会修改实例状态
public readonly double Sum()
{
return X + Y; // 只读取值，不修改状态
}
// 编译错误：readonly 方法试图修改实例状态
public readonly void Wrong()
{
X = 100; // 错误！不能修改字段
}
}

ref readonly​

ref return 上的 readonly 修饰符， 表示返回的引用不能被修改.

private int b = 10;
public ref readonly int GetReadOnlyRef()
{
    int x = 10;
   // b =20; // 错误！不能修改字段
    return ref x; // 返回 x 的引用
}

ref​

ref 关键字常用方式

  • 作为方法的参数， 表示该参数是引用传递， 可以修改实参的值, 外部传入的参数也会被修改
public void Swap(ref int a, ref int b)
{
    int temp = a;
    a = b;
    b = temp;
}
  • 作为返回值， 表示该方法返回的是引用类型， 可以修改实参的值
public ref int GetRef()
{
    return ref a;
}
  • 作为 ref 局部变量， 表示该变量是引用传递， 可以修改实参的值
int a = 10;
ref int b = ref a;
a = 20;
Console.WriteLine(b); // 输出：20
  • 作为 ref 条件表达式 或条件运算符 返回的是引用，类似变量 ref
ref int a = ref a > b ? ref a : ref b;
  • 用于struct ref struct 主要用于高性能场景，特别是在处理大量数据或需要严格控制内存分配的情况下, 只能在栈上分配。 有很多限制条件

  1. 不能出现在以下位置：

    • 类或普通结构体的字段
    • 接口的声明
    • 数组的元素
    • 装箱操作
    • lambda 表达式或本地函数中
    • async 方法中
    • 迭代器方法中

  2. 不能：

    • 实现接口
    • 作为泛型类型参数
    • 在堆上分配
    • 作为捕获变量
public ref struct Point
{
    public int X { get; set; }
    public int Y { get; set; }
}
  • 用于泛型约束 allows ref struct
class RefStructGeneric<T, S>
    where T : allows ref struct
    where S : T
{
    // etc
}

sizeof​

获取类型或变量的大小， 以字节为单位。 必须是非托管类型的参数，包括数值类型 枚举类型 元祖和结构类型。

int size = sizeof(int);
表达式常量值（字节）
sizeof(sbyte)1
sizeof(byte)1
sizeof(short)2
sizeof(ushort)2
sizeof(int)4
sizeof(uint)4
sizeof(long)8
sizeof(ulong)8
sizeof(char)2
sizeof(float)4
sizeof(double)8
sizeof(decimal)16
sizeof(bool)1

对于结构体

  1. 结构体的大小不是简单的成员大小之和
  2. 涉及内存对齐（Memory Alignment）
  3. 默认对齐大小为结构体中最大基本类型的大小
  4. 不包含引用类型大小，只包含引用类型的引用大小(4 或 8 字节)

内存对齐规则

// 示例1：简单结构体
struct Point
{
public int x; // 4字节
public int y; // 4字节
}
// sizeof(Point) = 8字节
// 示例2：不同类型混合
struct Person
{
public byte age; // 1字节
public int id; // 4字节
public short grade; // 2字节
}
// sizeof(Person) = 8字节（不是7字节！）
// 示例3：字段顺序影响大小
struct BadLayout
{
public byte b; // 1字节
public int i; // 4字节
public byte b2; // 1字节
}
// sizeof(BadLayout) = 12字节（因为填充）
// 示例4：优化字段顺序
struct GoodLayout
{
public int i; // 4字节
public byte b; // 1字节
public byte b2; // 1字节
}
// sizeof(GoodLayout) = 8字节
  1. 每个字段都会按照自身的大小或默认对齐大小进行对齐
  2. 结构体的总大小必须是最大对齐大小的整数倍
  3. 字段之间可能会插入填充字节

One more thing​
  1. as is <>强制类型转换 typeof 的区别
  • as 用于将一个对象转换为指定类型或可为 null 的类型。如果转换失败，返回 null。
  • is 用于判断一个对象是否可以转换为指定类型。
  • <> 强制类型转换用于将一个对象转换为指定类型。如果转换失败，会抛出异常。
  • typeof 用于获取一个类型的类型对象。
  • 优先使用 is 进行类型检查
  • 对于可能失败的引用类型转换使用 as
  • 确定会成功的转换使用强制转换
  • 需要类型信息时使用 typeof
  1. 协变（Covariance）和逆变（Contravariance）
  • 协变： 协变允许你使用比原始指定的类型更派生的类型。只能作为输出参数(out 关键字)
// 协变示例
IEnumerable<string> strings = new List<string>();
IEnumerable<object> objects = strings; // 允许，因为 IEnumerable<T> 是协变的

// 定义协变接口
interface ICovariant<out T>
{
    T GetItem();  // 只能作为返回类型
    // void SetItem(T item);  // 错误！协变类型参数不能用作输入参数
}
  • 逆变： 逆变允许你使用比原始指定的类型更通用（更基础）的类型。只能作为输入参数(in 关键字)
// 逆变示例
Action<object> objectAction = (object obj) => Console.WriteLine(obj);
Action<string> stringAction = objectAction; // 允许，因为 Action<T> 是逆变的

// 定义逆变接口
interface IContravariant<in T>
{
    void SetItem(T item);  // 只能作为输入参数
    // T GetItem();  // 错误！逆变类型参数不能用作返回类型
}
  1. virtual 和 abstract 的区别
特性virtualabstract
实现方式必须有方法体不能有方法体
重写要求可选重写必须重写
所属类型普通类抽象类
实例化类可以直接实例化类不能直接实例化

既然 virtual 可以有方法体， 为什么还要 abstract 呢？和普通基类中方法，子类中重写有什么区别？ 主要是在多态性上

特性普通方法（无 virtual）virtual 方法
运行时行为基于引用类型调用基于实际对象类型调用
子类覆盖方式只能使用 new（隐藏）使用 override（重写）
多态支持不支持支持
性能影响直接调用，性能较好需要查找虚表，性能略低
class BaseClass
{
    public virtual void Display()
    {
        Console.WriteLine("Base class");
    }

    public void Display2()
    {
        Console.WriteLine("Base class");
    }
}

class AClass : BaseClass
{
    public override void Display()
    {
        Console.WriteLine("A class");
    }

    public void Display2()
    {
        Console.WriteLine("A class");
    }
}

BaseClass baseObj = new BaseClass();
AClass aObj = new AClass();

baseObj.Display(); // 输出：A  class    有多态
aObj.Display(); // 输出：A class

baseObj.Display2(); // 输出：Base class   无多态
aObj.Display2(); // 输出：A class
