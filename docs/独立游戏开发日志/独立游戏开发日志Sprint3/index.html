独立游戏开发日志Sprint3

这个sprint delay了一天，依然是在搞基础脚手架相关的。

  1. 前端框架重构了一下ECS相关模块，数据可以双向绑定。

  2. 增加了多语言界面组件

  3. 增加游戏loading启动流程。buildin首包loading界面，加载热更包,本地存储，多语言，数值表。在Android上跑起来loading流程，AOT泛型问题之前没具体研究过，整了个分包策略，首包只包含loading界面是buildin的也可热更，其他包纯热更不含在包内，手机上有问题热更包里的代码被裁剪了，整了两天才搞好。。。

  4. 增加本地数据持久化（开发用）。我的需求是 需要本地存储复杂的游戏数据，整套玩家数据，可增量存储，也需要开关或者简单配置这种简单本地存储需求，所以没用又对比了下几个方案

  • unity的PlayerPrefs。无法满足全量玩家数据。

  • 二进制直接存储,自己写文件读取。只能整体存储，无法增量修改。频繁修改的话IO过于频繁了就。

  • SqlLit/LitDB这种嵌入式的本地数据库。高频写入感觉性能有点问题，有点重了。

  • 成品的EasySave3/SaveGamePro。似乎都不维护了，基础功能可以，但是数据大一点，高频一点感觉不靠谱。

  • 微信的MMKV。基于mmap内存映射的key-value组件，性能强又稳定，可高频写入可增量，微信在用，还一直在维护，github上有其他语言转化使用MMKV的，还没有c#或者unity直接可以用的。后续看情况桥接一下。

目前用一个开源的dre0dru/LocalStorage, 暂时用来做本地存储数据，开发用先跑起来，使用它的文件存储方法，然后整存整取二进制文件，序列化和反序列化暂时简单用json.
